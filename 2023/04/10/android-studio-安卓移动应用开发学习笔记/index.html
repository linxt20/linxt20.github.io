android studio 安卓移动应用开发学习笔记

发表于2023-04-10|更新于2023-04-10|技术android
|字数总计:3.7k|阅读时长:13分钟|阅读量:

这篇学习笔记主要基于B站up主Android开发进阶的教学视频，同时作者也在其中加入了自己的一些心得理解与技巧总结

第一章 控件

1、TextView

  • 基础属性：

    • layout_width:组件的宽度 单位是dp
    • layout_margintop:距离上面的组件的距离
    • layout_height:组件的高度
    • id:为控件设置一个id，可以在java文件当中使用 控件 varname = findviewbyid(R.id.xxx)的方式获取控件
    • text:控件显示的文本内容
    • textColor:控件文本的颜色设置，总共八位，前两位为透明度，3-4位是红色，5-6位是绿色，7-8位是蓝色
    • textStyle:设置字体风格，默认normal，可选bold(加粗)，italic(斜体)
    • textSize: 设置字体大小 单位是sp
    • background: 设置控件的背景，可以设置颜色(八位数字)，也可以设置图片、xml等
    • backgroundTint: 设置控件的背景的颜色，可以给原有的图标背景上色，只具有着色功能
    • foreground: 设置控件的前景，注意会覆盖text
    • gravity: 设置控件中内容的位置，可选center_horizontal,center,center_vertical,bottom,right,clip_horizontal,clip_vertical,end,fill,fill_horizontal,fill_vertical,left等

  • 阴影属性:(需要四个一起配合使用)

    • shadowColor: 设置阴影的颜色
    • shadowRadius: 设置阴影的模糊宽度，一般选择3.0
    • shadowDx: 设置阴影在水平方向的偏移度
    • shadowDy: 设置阴影在竖直方向的偏移度

  • 跑马灯效果(内容需要超出width)

    • singleline:内容单行显示(超出部分内容显示省略号)
    • focusable:是否可以获取焦点(在有焦点的情况下运行)
    • focusableInTouchMode:触摸状态下是否可以聚焦
    • ellipsize:在哪里省略文本(显示省略号的位置或者没有省略号)
    • marqueeRepeatUmit:字幕动画重复的次数

2、Button空间

  • 基础属性
    • button的类实现是继承于TextView的，因此TextView的属性button都有
  • 三个点击属性：
    • 执行顺序：点击按钮，触发touch 进入，longclick，移动触发 touch 移动，放开按钮 触发 touch 离开，click
    • 当ontouch返回true的时候，longclick和click都不会执行；当longclick返回true的时候，ontouch会执行，而click不会执行

3、draw资源文件

  • 创建一个新的draw，在drawable文件夹右键新建一个资源文件，资源文件默认是sector标签作为根的，给其中加上内容就需要使用item标签
  • 一个item中可以使用属性:
    • color:颜色(一般颜色选择器会重新创建一个文件夹) 常用于设置控件的backgroundTint属性
    • drawable:引用的Drawable位图，注意不能是颜色 常用于设置控件的background属性
    • state_focused:是否获得焦点
    • state_pressed:控件是否被按下
    • state_enabled:控件是否可用
    • state_selected:控件是否被选择,针对有滚轮的情况
    • state_checked:控件是否被勾选
    • state_checkable:控件可否被勾选,eg:checkbox

4、edittext

  • 基础属性
    • button的类实现是继承于TextView的，因此TextView的属性button都有
    • hint: 输入提示文本
    • textcolorhint: 提示文本的颜色
    • inputtype: 输入的类型
    • drawableleft/drawableright: 输入框指定方位添加图片
    • drawablepadding: 图片与输入框之间的距离
    • paddingleft/right/top/bottom: 设置输入文本与输入框之间的距离
  • 获取文本内容的方式: 先用findviewbyid找到edittext，然后用edittext的 .gettext().toString()操作即可

5、 imageview

  • 基础属性:
    • src:设置图片资源
    • scaleType:设置图片填充方式，fitstart/center/end(等比例缩放后放置于左上角/中间/右下角),fitXY(图片横纵方向适应imageview),center(保持原图的大小，显示在mageView的中心。当原图的size大于imageView的size，超过部分裁剪处理),centerCror(保持宽高比缩放图片，直到完全覆盖ImageView，可能会出现图片的显示不完全),centerInside(保持宽高比缩放图片，直到ImageView能够完全地显示图片),matrix(不改变原图的大小，从lmageView的左上角开始绘制原图，原图超过mageView的部分作裁剪处理)
    • maxHeight: 最大高度
    • maxWidth: 最大宽度
    • adjustViewBounds: 调整View的界限,选择true就会按照前面的两个max来限制图片的大小

6、processbar

  • 基础属性(下面四个都不设置就是一个圆形的加载进度条)
    • max: 进度条的最大值
    • process: 进度条已经完成的进度值
    • indeterminate: 设置成true，进度条不精确显示进度
    • style=”?android:attr/progressBarStyleHorizontal” 水平进度条 (需要同时加上max的设置)
  • 设置显示和隐藏
    • 点击函数 public void onclick(view v)
    • 获取显示状态 getVisibility() 值为 v.GONE 和 v.VISIBLE
    • 设置显示状态 setVisibility(v.GONE) setVisibility(v.VISIVLE)
  • 设置进度条进度
    • 获取当前进度条进度 getProcess() 为int类型
    • 设置当前进度条进度 setProcess(10)

7、notification和notificationmanager

  • 通知渠道的重要性可选项如下：
    • IMPORTANCE_NONE 关闭通知
    • IMPORTANCE_MIN 开启通知，不会弹出，但没有提示音，状态栏中无显示
    • IMPORTANCE_LOW 开启通知，不会弹出，不发出提示音，状态栏中显示
    • IMPORTANCE_DEFAULT 开启通知，不会弹出，发出提示音，状态栏中显示
    • IMPORTANCE_HIGH 开启通知，会弹出，发出提示音，状态栏中显示
  • 通知的常见设置：
    • setContentTitle(String string) 设置标题 *这个是必须设置的
    • setContentText(String string) 设置文本内容 *这个是必须设置的
    • setSmallcon(int icon) 设置小图标 *这个是必须设置的
    • setLargelcon(Bitmap icon) 设置通知的大图标
    • setColor(int argb) 设置小图标的颜色
    • setContentIntent(PendingIntent intent)设置点击通知后的跳转意图
    • setAutoCancel(boolean boolean) 设置点击通知后自动清除通知
    • setWhen(long when) 设置通知被创建的时间

8、toolbar

  • toolbar是指app的标题栏,但是app原有的样式中就已经有一个标题栏是actionbar，如果要使用toolbar替换actionbar，需要在res/values文件夹中把themes.xml中style的parent从Theme.MarterialComponents.DayNight.DarkActionBar修改为Theme.MarterialComponents.DayNight.NoActionBar
  • 在activity_main.xml中使用toolbar，需要注意应该使用的是androidx.appcompat.widget.Toolbar而不是Toolbar
  • 需要注意toolbar在java文件中自动导包的时候，需要注意不是import android.widget.Toolbar 而是 import androidx.appcompat.widget.Toolbar
  • toolbar的layout_width为match_parent，layout_height = “?attr/actionBarSize”表示和原来的actionbar一样高
  • 基础属性(需要注意下面这些属性开头是app:而不是android:，因为这个用的不是android自带的toolbar)
    • navigationIcon:设置icon图标，常用于设置返回箭头icon图标,点击监控函数需要只用 name.setNavigaionOnClickListener(new View.OnClickListener(){ @override public void onclick(view v){ } });
    • title:主标题
    • titleTextColor:主标题的颜色
    • titleMarginStart:设置标题距离左边的距离
    • subtitle:子标题(在主标题下面，字体小一点)
    • subtitleTextColor:子标题的颜色
    • logo:也是一个图标 toolbar从左到右为navigationIcon logo title

第二章 布局

1、linearlayout

  • 常见属性
    • orientation 布局中组件的排列方式
    • gravity 空间组件所包含的子元素的对齐方式，可以多个组合 例如 center_horizontal|bottom 设置水平居中对齐，同时与底边对齐
    • layout_gravity 控制该组件在其父容器里面的对齐方式
    • background 背景图、背景色
    • divider 设置分割线的样式，常为一个分割线图片 单独设置没有效果，需要设置showDividers
    • showDividers 设置分割线所在的位置 none 无 beginning 开始 end 结束 middle每两个组件之间
    • dividerPadding 设置分割线的padding 简单的例子，当linearlayout设置为vertical的时候，这个时候的padding是指到左右边界的距离
    • layout_weight 分配剩余区域占比权重 注意是在原有长度的基础上，增加分配剩余空间的占比;需要注意，如果其中有一个元素使用了match_parent，那么其他区域的权重将会失效，如果三个元素都用了match_parent，权重分别为2 1 1，此时剩余空间为 1-3 = -2 那么权重为2的占据屏幕的比例是在原有的1的基础加上 -2*(2/4) = -1 也就是此时占比为0，其他两个可以同理计算得到0.5，也就是此时权重2的不显示，权重1的各占一半

2、relativelayout

  • 默认都是相对于父容器左上角开始布局
  • 常见属性
    • 根据父容器定位 设置为true即为设置成功
      • layout_alignParentLeft 左对齐
      • layout_alignParentRight 右对齐
      • layout_alignParentTop 顶部对齐
      • layout_alignParentButtom 底部对齐
      • layout_centerHorizontal 水平居中
      • layout_centerVertical 竖直居中
      • layout_centerInParent 中间位置
    • 根据兄弟组件定位 设置参考组件的id即可 简单的例子 如果目标组建的id=“@+id/xx” 那么设置参考组件的时候可以设置为 “@id/xx”
      • layout_toLeftOf 放置于参考组件的左边
      • layout_toRightOf 放置于参考组件的右边
      • layout_above 放置于参考组件的上方
      • layout_below 放置于参考组件的下方
      • layout_alignTop 对齐参考组件的上边界
      • layout_alignBottom 对齐参考组件的下边界
      • layout_alignLeft 对齐参考组件的左边界
      • layout_alignRight 对齐参考组件的右边界
    • margin 设置组件和父容器的边距
      • layout_margin( / Left / Right / Top / Bottom) 上下左右偏移
    • padding 设置组件与内部元素的边距
      • layout_padding( / Left / Right / Top / Bottom) 上下左右偏移

3、framelayout(少用)

  • 这个布局是从左上角开始，按照组件定义的顺序依次画出相应组件，后画的会覆盖先画的
  • 常用属性：
    • foreground 前景色
    • foregroundGrivity 前景色/前景图的对齐位置

终章 部分技巧

1、在activity_main.xml中删除原有的代码，加上linearlayout，随后树屋width回车即可出现xmlns相关代码，使用ctrl+alt+L可以使xmlns的代码放在最上面。

2、对于控件属性的取值，按住ctrl再去点击属性，即可跳转到属性取值的代码部分

3、布局文件中的文本、颜色需要放在res资源文件下的values文件中的colors.xml和string.xml中设定，然后在布局文件中调用@string或者@color

4、重写TextView:创建一个类，然后增加extends TextView,然后鼠标点击类名，点击create，选择需要重写的函数进行重写；在activity_main.xml中调用使用 重写textview的java文件的包名+文件名(不包含后缀),例如: com.enjoy.mytextview.mytextview

5、在button控件中直接使用background设置button的背景色是不会成功的，这是因为版本更新之后出现的问题，需要在res资源文件夹下的values文件夹中的themes.xml中修改style的parent属性，在最后加上.Bridge后即可实现颜色修改。

6、打印函数Log.e(TAG,”msg”); 程序运行之后在logcat中找到输出

7、如果想要让toolbar标题居中，可以不设置title属性，直接在toolbar标签中间创建一个textview，给其layout_gravity属性设置为center即可

8、layout_gravity属性和gravity属性的区别:gravity是控制一个视图内部的子元素如何相对于该视图进行对齐；layout_gravity确定一个视图或视图组件如何相对于其父视图进行对齐

9、设置分割线，可以在元素中间加上如下的代码 <view android:background = "#00ff00" android:layout_width = "match_parent" android:height = "1dp" />
